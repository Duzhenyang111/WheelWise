# External Skills and Reference Resources

WheelWise may reference or optionally depend on external skill ecosystems, but it must not copy their content into this repository.

## UI UX Pro Max

UI UX Pro Max is the design intelligence reference for ui-demo, visual-brief, and report-visualization work.

- GitHub: https://github.com/nextlevelbuilder/ui-ux-pro-max-skill
- Skills listing: https://skills.sh/nexu-io/open-design/ui-ux-pro-max
- Suggested use: design intelligence reference for `ui-demo`, `visual-brief`, and `report-visualization`, including UI direction, information architecture, design-system thinking, style selection, component/layout reasoning, stateful prototype planning, responsive behavior, and anti-pattern review.
- WheelWise role: pass product type, user, surface, modules, constraints, product strategy, prototype needs, report visualization needs, and asset needs to a UI/UX skill when visual, prototype, or `index.html` planning is needed.
- Boundary: do not copy UI UX Pro Max content into WheelWise. Capture only WheelWise-specific adaptation notes and final decisions.
- Webpage display: UI UX Pro Max may guide layout, hierarchy, color, typography, visualization module choice, and anti-pattern checks for `index.html`, but the webpage content must still come from WheelWise's Chinese `report.md`.
- WheelWise adaptation: report pages should avoid one-note blue SaaS card pages, generic marketing heroes, nested cards, low-density summaries, and static prototypes. Prefer surface-aware layouts, matrices, timelines, flows, decision boards, stateful controls, and responsive constraints that match the product type.

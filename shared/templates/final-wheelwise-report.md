# 报告说明与阅读导览

输出文件夹：`wheelwise-report/`

源报告：`report.md`

网页展示：`index.html`

交互原型：`prototype.html`

内部状态：`project-state.md`

证据中枢：`evidence-board.md`

报告目的：
用较短篇幅说明产品想法的核心判断、交付形态、复用决策、视觉与演示方向和下一步执行计划。

适用阶段：
适用于用户要求快速评估、轻量验证或需要一份可继续扩写的中文报告初稿。

核心结论预览：
本报告先给结论，再按用户问题、方案判断、视觉与演示、风险验证、执行计划和行动建议递进展开。

阅读方式：
先看结论和交付形态，再看自研 / 购买 / 复用 / 分叉改造 / 参考决策，最后看可交给 Codex 执行的计划与继续 / 停止条件。

## 可交给 Codex 执行的计划

源报告任务：

```text
生成或更新 wheelwise-report/report.md，确保报告正文全中文、结构递进，并包含视觉说明、交互演示、网页展示文件记录和最终行动建议。
```

网页展示文件任务：

```text
如启用展示层，生成或更新 wheelwise-report/index.html，并确保它是来自 report.md 的报告可视化解释层，包含导航、核心结论、视觉模块、原型入口、响应式布局和主要报告内容覆盖。
```

交互原型任务：

```text
生成或更新 wheelwise-report/prototype.html，使用本地模拟数据模拟目标交付形态，包含点击、输入、状态变化、加载 / 空状态 / 错误 / 成功状态和未接入真实后端说明。
```

内部状态与证据任务：

```text
生成或更新 wheelwise-report/project-state.md 与 wheelwise-report/evidence-board.md，确保 Gate 状态、关键假设、证据、缺口和下一步动作可追踪。
```

# Final Output Contract

WheelWise final output must include:

- A direct verdict: Build MVP, Validate First, Pause, or Reject.
- A decision rationale summary using `../../shared/references/decision-rationale-standard.md`.
- The selected delivery surface and why alternatives lost.
- Product strategy: positioning, differentiation, MVP scope, user-facing workflow, feature priority, product wedge, and validation focus.
- A module-level Build / Buy / Reuse / Fork / Reference matrix.
- A technical implementation path that follows the selected surface.
- A visual brief with image-level visual assets or production prompts.
- A UI demo or interaction demo plan. For API/CLI/automation products, use a playground, terminal simulator, request explorer, or workflow simulator.
- Key risks with mitigations.
- Validation experiments before or during build.
- A Codex-ready execution plan with milestones, tasks, files or components, tests, acceptance criteria, visual-brief tasks, and demo tasks.

Use `../../shared/templates/new-product-brief.md` for full product planning output.
